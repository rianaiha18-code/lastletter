最終更新日：2026/07/18

# 1. システム概要

## システム名

Last Letter

## 目的

Last Letterは、自分が亡くなった後に家族や大切な人へメッセージや希望を残すことができるWebアプリケーションである。

メッセージだけでなく、

- 葬儀の希望
- デジタル資産
- 大切な人の連絡先

などを一元管理し、遺族の負担軽減を目的としている。

---

# 2. システム構成

```text
                Browser
        (HTML / CSS / JavaScript)
                    │
          HTTP Request / Response
                    │
                    ▼
       Render（Node.js / Express）
                    │
              REST API
                    │
                    ▼
          Railway（MySQL）
```

# 6. 機能一覧（完成イメージ）

家族や友人へ想いを残す。

# 11. 開発メモ

## Render

Node.jsアプリを公開するために利用。

GitHubへpushすると自動デプロイ。

---

## Railway

MySQLデータベースをクラウド上で運用。

Renderから接続。

---

## 工夫した点

- ローカルストレージではなくMySQLへ保存
- Express APIによるCRUD
- セッション管理
- パスワードハッシュ化
- デモアクセス機能
- ユーザーごとのデータ管理
